# 企微自动打标需求 - 需求采集记录 v1

记录时间：2026-05-18

## 背景

现有会员小程序基于消费者业务中台的分群规则能力，支持业务设置不同条件创建分群规则，并自动对会员进行分层。会员进入小程序时，系统会识别其所属分群，并展示对应广告内容。目前分群规则主要应用在展示层。

业务希望将分群规则统计出来的用户，同步应用到企业微信客户运营场景：对于指定分群命中的企微客户，在企微侧打上指定企业微信标签，便于业务后续针对指定客户群推送内容与开展运营举措。

## 已确认范围

1. 分群规则编辑新增「统计频率」。
   - 可选：每天统计一次、仅统计一次。
   - 默认不选中，保存时必选。
2. 只有「仅统计一次」且「最后统计时间为空」的分群规则，才在列表操作区展示「企微打标任务」按钮。
3. 企微打标任务绑定在分群规则上。
4. 点击「企微打标任务」进入独立页面，可选择企微标签、保存任务配置、查看任务状态。
5. 企微标签数据源来自现有企微自建系统，通过接口读取。
6. 每条分群规则只能选择一个企微标签。
7. 分群规则未执行统计前，可修改或删除已配置的企微标签。
8. 分群规则已触发统计后，不允许再修改企微标签。
9. 企微打标任务状态：未配置、待执行、已执行。
10. 分群规则用户列表新增「企微打标状态」。
    - 待打标
    - 无需打标
    - 打标成功
    - 打标失败
11. 未设置企微打标任务的分群规则，用户记录打标状态为「无需打标」。
12. 打标失败记录展示失败原因，原因包括：未匹配到企微客户、接口调用失败、其他系统异常。
13. 分群规则用户列表支持勾选打标失败用户后批量重试。
14. 新增分群条件：
    - 属性 / 通用属性：生日时间
    - 属性 / 会员属性：推荐官身份
    - 行为 / 线上订单：订单实付金额
    - 行为 / 线下订单：安装时间、运营商主体

## 统计频率与生日时间交互补充（2026-05-18）

### 统计频率
- 在分群规则编辑页中，统计频率配置项放在「规则状态」下方。
- 可选项：每天统计一次、仅统计一次。
- 默认不选中。
- 保存时必填。

### 生日时间
入口：点击「添加属性」后，属性区域新增一行。

属性条件行：
1. 第一个下拉框选择属性分类：通用属性 / 会员属性。
2. 当选择「通用属性」时，右侧字段下拉框可选：位置、用户身份、分销员身份、生日时间。
3. 当选择「生日时间」后，右侧选项条件只支持「指定月日范围」。
4. 当选择「指定月日范围」时，右侧出现开始月日、结束月日；起止均只选择月和日，不选择年份。
5. 指定月日范围的统计口径以后续在线文档描述为准；原型仅表达前端配置项。

对应原型：`原型/rule-edit-v0.1.html`

## 分群条件交互补充（2026-05-18 18:38）

### 通用规则
- 属性、行为支持创建多条。
- 标签只允许创建一条，本版本不重点输出标签原型。
- 不同类型之间统一受顶部「必须同时满足 / 满足其一即可」控制。
- 每行条件都支持删除。
- 同一个条件不允许重复添加，沿用现有系统校验，本版本 PRD 可不重点展开。

### 推荐官身份
- 点击添加属性，在属性区域新增一行。
- 第一个下拉框选择「会员属性」。
- 右侧属性下拉框选择「推荐官身份」。
- 再右侧选项条件下拉框选择「等于」。
- 右侧展示两个单选项：「推荐官」「非推荐官」。

### 线上订单：主单订单金额
- 点击添加行为，在行为区域新增一行。
- 第一个下拉框选择「线上订单 -> 下单」。
- 右侧属性下拉框选择「主单订单金额」。
- 右侧选项条件支持：指定范围、等于、小于、大于。
- 选择「指定范围」时，展示两个金额输入框，填写起止金额区间。
- 选择「等于 / 小于 / 大于」时，展示一个金额输入框。

### 线下订单：安装时间
- 点击添加行为，在行为区域新增一行。
- 第一个下拉框选择「线下订单 -> 下单」。
- 右侧属性下拉框选择「安装时间」。
- 右侧时间选项交互与生日时间一致：固定起止时间 / 相对时间范围。

### 线下订单：运营商主体
- 点击添加行为，在行为区域新增一行。
- 第一个下拉框选择「线下订单 -> 下单」。
- 右侧属性下拉框选择「运营商主体」。
- 右侧条件选项支持：包含任一、包含全部、不包含。
- 右侧展示选中框，框内有添加按钮。
- 点击添加按钮显示弹窗。
- 弹窗支持输入运营商编码；输入后校验按钮亮起。
- 点击校验按钮调用接口查询；若查询到结果，下方展示运营商名称。
- 点击确认后，把运营商名称和编码展示在选中框内，格式：深圳市索菲亚(Y1234)。
- 选中框内每个运营商右侧有删除 icon，可删除。
- 支持通过相同方式添加多个运营商。

对应原型：`原型/rule-edit-v0.4.html`
